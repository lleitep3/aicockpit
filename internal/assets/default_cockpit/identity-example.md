# Developer Identity Example

> This file is an example identity file for AICockpit.
> Copy it to `identity.md` and update the values for your own developer profile.

## About Me

- **Name:** Your Name
- **Role:** Your role or title
- **Location:** Your city / timezone
- **Primary language:** Your preferred language for communication; English for code and docs

## Development Central

**This cockpit is my personal development hub.**

- All AI tools should treat this as the source of truth for my local setup
- When creating new projects, use my primary development workspace
- Use this identity file for project conventions, style rules, and AI behavior

## Architecture Preferences

- **API design:** Your API style
- **Async-first:** Your async / concurrency preference
- **Separation of concerns:** Your architecture principles
- **Small, focused functions:** Prefer composition and minimal functions
- **Fail fast:** Validate early and handle errors explicitly

## Work Style

- **TDD preferred:** Write tests early and keep them updated
- **Iterative:** Ship small, working increments
- **Documentation:** Document why, not what
- **Code review:** Review before committing
- **Commits:** Atomic, conventional commits

## Communication Preferences (for AI tools)

- **Be direct:** Skip preambles and get to the solution
- **Show, don't tell:** Prefer examples over long explanations
- **Ask before deleting:** Confirm before removing files or data
- **Portuguese or English:** Match the session language
