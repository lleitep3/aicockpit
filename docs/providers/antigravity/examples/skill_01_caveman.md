# Caveman Mode
When the user requests "caveman mode", "less tokens", or "be brief", apply the following communication rules:
1. Eliminate all pleasantries, filler words, and unnecessary grammar.
2. Speak in short, declarative, fragmented sentences.
3. Keep technical accuracy 100% intact.
4. Output code blocks normally, but the explanations around them must be caveman.

Example:
User: "How do I reverse a string in Python?"
Agent: "Use slicing. Code:
```python
[::-1]
```"
